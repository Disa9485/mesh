Lets add simple physics. In view mode only lets have a button to toggle physics on/off, add this after the mesh buttons. We should implement a simple hooke law springs based system. We should take every vertex that has physics enabled and every edge that the vertex touches should become a spring. The physics enabled vertex should no longer respond to parameter's instead its position is based on live physics. Edges are a spring and their rest lengths are based on what their length when physics is enabled. When we alter a parameter we should not move these physics enabled vertices directly, the ones without physics will still move and anchor the soft vertices to the rest of the body. We should try to only create physics once, we should only rebuild the physics vertices if parameter distance jumps to far to prevent these springy edges from moving rapidly and breaking, so physics should still work if the user is slowly dragging a parameter slider but if they move it too quickly, lets say more than 0.25 in 1 second or if it jumps more than 0.1 instantly (i.e. they right click a setpoint) we should rebuild the physics bodies if the parameter layers with physics. We should only rebuild the partial softbodies related to that parameter if this happens, this just means setting the springs length/angles back to the lengths/angles where it is when physics is disabled. UV's should update live to move with the physics enabled meshes. For now just use a global physics settings but we will want physics settings tunable soon, so have them easily accessible. There is no need to introduce collision yet, lets just have simple soft bodies effected by gravity and forces from other rigid vertices (rigid vertices are effected by parameters though).
